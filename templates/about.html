About

What is this?

Scheduling Gauntlet is a site for practicing process scheduling questions. Users are presented with scheduling problems which can be solved for points. Users with the most points are displayed on a global leaderboard.

Pog. Why?

For fun.

Scheduling Gauntlet was also initially created as tool to generate personal practice problems for a course I was taking.

How does the game work?

You can join and compete against other users by signing up for an account.

Once logged in, you can get a new scheduling problem by pressing the "Challenge" button. Solving the problem correctly will earn you 1 point. There is no penalty for solving a problem incorrectly (blank answers effectively skip the question).

How do I solve a scheduling problem?

A scheuling problem is comprised of two main parameters: the scheduling method and the process arrival and execution times.

Scheduling Method

The scheduling method describes the strategy for determining which process gets to be executed at given points in time. Below are a list of strategies that can be encountered in Scheduling Gauntlet:

  • First-come, first-served (FCFS): Processes are executed in the order that they arrive. Processes enter a queue on arrival. When an executing process finishes, the next process to be executed is the next process in line in the queue.
  • Shortest job first (SJF): Processes are executed based on shortest execution time. When an executing process finishes, the next process to be executed is the process with the shortest execution time among the processes that have already arrived.
  • Shortest remaining time first (SRTF): Processes are executed based on shortest execution time left. When the executing process finishes or another process arrives, the next process to be executed is the process with the shortest execution time left among the arrived processes. This method has the potential to pause and resume process execution.
  • Round robin (RR): Processes are executed and rotated based on a short period of time (quantum). Processes are placed in a queue as they arrive. If the quantum for the currently executing process expires before the process finishes, that process is paused, moved to the back of queue, and the next process in the queue is picked up and started/resumed.

Process Times

Problems also specify a list of processes, each with an arrival and execution time. A scheduling problem is successfully solved if the the correct finish and wait times for each process are submitted. Below are brief descriptions for each type of time:

  • Arrival: The time at which a process arrives and is available to be executed.
  • Execution: The amount of time a process needs to execute before finishing.
  • Finish: The time at which a process finishes execution.
  • Wait: The amount of time a process spends not executing after arriving.
⚠️ NOTE: Sometimes problems generate with ties for which process should be executed next (ex. process arrives with same remaining time as current executing process when using SRTF). Ties are broken in Scheduling Gauntlet by giving precedence to processes with the lowest process number.
